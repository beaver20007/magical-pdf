Magical PDF: работа на Windows ПК

Главная идея: GitHub хранит общий проект, а каждый компьютер берёт последнюю версию оттуда и отправляет обратно законченные изменения.

1. Что где хранится

Общий репозиторий проекта:

https://github.com/beaver20007/magical-pdf

В репозитории хранятся исходники WEB, Windows, macOS и iOS версий. Готовые installer-файлы .exe, .dmg и временные папки сборки в Git не кладём.

2. Один раз подготовить Windows ПК

  1. Установите Git: https://git-scm.com/download/win
  2. Установите Node.js LTS: https://nodejs.org/
  3. Установите Rust: https://www.rust-lang.org/tools/install
  4. Установите Visual Studio Build Tools 2022.
  5. В Visual Studio Build Tools выберите workload Desktop development with C++, MSVC v143 и Windows 10/11 SDK.
Если PowerShell пишет, что npm.ps1 заблокирован политикой выполнения сценариев, используйте npm.cmd вместо npm. Это нормальный Windows-обход без изменения системной политики.

3. Первый раз скачать проект

Откройте PowerShell и выполните:

cd $env:USERPROFILE\Documents
git clone https://github.com/beaver20007/magical-pdf.git
cd magical-pdf
npm.cmd ci

После этого папка проекта будет здесь:

%USERPROFILE%\Documents\magical-pdf

4. Каждый раз перед началом работы

Перед тем как что-то менять, подтяните свежую версию из GitHub:

cd $env:USERPROFILE\Documents\magical-pdf
git switch main
git pull origin main
npm.cmd ci
Если команда git status после этого ничего не показывает, значит локальная папка чистая и совпадает с GitHub.

5. Запустить WEB-версию для проверки

npm.cmd run dev:web

Откройте в браузере:

http://127.0.0.1:5173/

6. Собрать Windows installer

npm.cmd run build:windows

Готовый installer появится здесь:

src-tauri\target\release\bundle\nsis\

Например:

Magical PDF Listva_0.1.0_x64-setup.exe
Этот .exe не нужно добавлять в Git. Его можно отправить отдельно, приложить к GitHub Release или хранить как artifact сборки.

7. Как сохранить свою работу в GitHub

Простой вариант для небольших изменений

git status
git add .
git commit -m "Короткое описание изменения"
git push origin main

Аккуратный вариант для заметной задачи

git switch -c feature/short-task-name

# внесите изменения и проверьте приложение

git status
git add .
git commit -m "Describe the change"
git push -u origin feature/short-task-name

После push откройте Pull Request на GitHub и объедините его в main.

8. Как продолжить с другого компьютера

  1. На старом компьютере закончите работу: git status, git add, git commit, git push.
  2. На новом компьютере откройте проект.
  3. Выполните git pull origin main.
  4. Продолжайте работу с того же состояния.

9. Быстрая диагностика

git status
node --version
npm.cmd --version
rustc --version
cargo --version

Если rustc или cargo не находятся, закройте PowerShell и откройте заново. Если не помогло, проверьте установку Rust.

10. Главное правило

Перед работой: git pull origin main. После законченной работы: git commit и git push. Тогда проект всегда можно открыть на любом компьютере и продолжить с последнего общего состояния.

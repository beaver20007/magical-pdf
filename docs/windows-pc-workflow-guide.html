Magical PDF: простая работа на Windows ПК

Эта инструкция написана для обычного сценария: открыть общий проект, обновить его, выбрать папку в Codex и продолжить работу.

1. Главное, что нужно запомнить

У проекта есть одно общее место хранения:

https://github.com/beaver20007/magical-pdf

На каждом компьютере лежит своя копия папки magical-pdf. Перед работой её нужно обновить из GitHub. После работы изменения нужно отправить обратно в GitHub.

Новый удобный способ: запускать файл scripts\start-windows.cmd. Он сам обновляет проект и готовит его для Codex.

2. Что один раз должно быть установлено на Windows

  1. Откройте браузер.
  2. Установите Git: https://git-scm.com/download/win
  3. Установите Node.js LTS: https://nodejs.org/
  4. Для сборки Windows-приложения установите Rust: https://www.rust-lang.org/tools/install
  5. Для сборки Windows-приложения установите Visual Studio Build Tools 2022.
  6. В Visual Studio Build Tools выберите Desktop development with C++, MSVC v143 и Windows 10/11 SDK.
Если вы только работаете с Codex и не собираете installer, чаще всего достаточно Git и Node.js. Rust и Visual Studio Build Tools нужны именно для сборки .exe.

3. Если проекта ещё нет на этом ПК

  1. Откройте папку Документы.
  2. Щёлкните правой кнопкой мыши по пустому месту.
  3. Выберите Открыть в Терминале или Открыть окно PowerShell здесь.
  4. Скопируйте и вставьте команды ниже.
git clone https://github.com/beaver20007/magical-pdf.git
cd magical-pdf
scripts\start-windows.cmd

После этого проект будет лежать в папке:

C:\Users\ВАШЕ_ИМЯ\Documents\magical-pdf

4. Обычный старт работы на этом ПК

  1. Откройте Проводник.
  2. Перейдите в Документы.
  3. Откройте папку magical-pdf.
  4. Откройте папку scripts.
  5. Дважды щёлкните по файлу start-windows.cmd.
  6. Дождитесь надписи Ready.
  7. Не закрывайте окно сразу: в нём будет написан путь к папке проекта.

Скрипт сам делает три важные вещи:

  • переходит на основную ветку проекта main;
  • подтягивает последнюю версию из GitHub;
  • устанавливает зависимости через npm ci.

5. Как открыть проект в Codex

  1. Откройте Codex.
  2. Нажмите кнопку выбора папки или проекта.
  3. Выберите папку Documents\magical-pdf.
  4. Если Codex спросит подтверждение, подтвердите открытие этой папки.
  5. Напишите Codex такую фразу:
Продолжи работу над Magical PDF. Сначала проверь git status, затем помоги с задачей.
Если перед этим вы запускали start-windows.cmd, Codex откроет уже свежую версию проекта.

6. Как продолжить работу с другого компьютера

  1. На первом компьютере попросите Codex сохранить работу в GitHub.
  2. Убедитесь, что Codex написал, что push выполнен.
  3. На другом компьютере откройте папку magical-pdf.
  4. Запустите scripts\start-windows.cmd.
  5. Откройте эту же папку в Codex.
  6. Продолжайте работу.

Готовая фраза для Codex после завершения задачи:

Сохрани эти изменения в GitHub в общий проект Magical PDF.

7. Как запустить WEB-версию для проверки

  1. Откройте папку magical-pdf.
  2. Щёлкните правой кнопкой мыши по пустому месту.
  3. Выберите Открыть в Терминале.
  4. Выполните команду:
npm.cmd run dev:web

Откройте в браузере:

http://127.0.0.1:5173/

8. Как собрать Windows installer

  1. Откройте папку magical-pdf.
  2. Откройте терминал в этой папке.
  3. Выполните команду:
npm.cmd run build:windows

Готовый installer появится здесь:

src-tauri\target\release\bundle\nsis\
Файл .exe не добавляйте в Git. В Git хранятся исходники, а installer собирается отдельно.

9. Что делать, если что-то пошло не так

  • Если Codex видит старую версию проекта, запустите scripts\start-windows.cmd ещё раз.
  • Если Windows спрашивает разрешение на запуск скрипта, разрешите запуск.
  • Если npm не запускается, используйте npm.cmd.
  • Если сборка .exe не идёт, проверьте Rust и Visual Studio Build Tools.

10. Самая короткая памятка

1. Открыть Documents\magical-pdf\scripts
2. Запустить start-windows.cmd
3. Открыть Codex
4. Выбрать папку Documents\magical-pdf
5. Написать Codex, что нужно сделать
6. В конце попросить Codex сохранить изменения в GitHub

Magical PDF: простая работа на macOS

Эта инструкция для сценария: открыть общий проект на Mac, обновить его, выбрать папку в Codex и продолжить работу.

1. Главное, что нужно запомнить

У проекта есть одно общее место хранения:

https://github.com/beaver20007/magical-pdf

На каждом компьютере лежит своя копия папки magical-pdf. Перед работой её нужно обновить из GitHub. После работы изменения нужно отправить обратно в GitHub.

Новый удобный способ: запускать файл scripts/start-macos.command. Он сам обновляет проект и готовит его для Codex.

2. Что один раз должно быть установлено на Mac

  1. Откройте App Store.
  2. Установите Xcode.
  3. Откройте Xcode один раз и примите условия.
  4. Установите Node.js LTS: https://nodejs.org/
  5. Для сборки macOS-приложения установите Rust: https://www.rust-lang.org/tools/install
  6. Для iOS-сборки установите CocoaPods.
sudo gem install cocoapods
Для простой работы с Codex чаще всего достаточно Git, Node.js и открытой папки проекта. Xcode, Rust и CocoaPods нужны для сборок macOS/iOS.

3. Если проекта ещё нет на этом Mac

  1. Откройте приложение Terminal.
  2. Скопируйте и вставьте команды ниже.
  3. Нажмите Enter.
cd ~/Documents
git clone https://github.com/beaver20007/magical-pdf.git
cd magical-pdf
chmod +x scripts/start-macos.command
./scripts/start-macos.command

После этого проект будет лежать в папке:

~/Documents/magical-pdf

4. Обычный старт работы на Mac

  1. Откройте Finder.
  2. Перейдите в Documents.
  3. Откройте папку magical-pdf.
  4. Откройте папку scripts.
  5. Дважды щёлкните по файлу start-macos.command.
  6. Если macOS спросит разрешение на запуск, разрешите.
  7. Дождитесь надписи Ready.

Скрипт сам делает три важные вещи:

  • переходит на основную ветку проекта main;
  • подтягивает последнюю версию из GitHub;
  • устанавливает зависимости через npm ci.

5. Как открыть проект в Codex

  1. Откройте Codex.
  2. Нажмите кнопку выбора папки или проекта.
  3. Выберите папку Documents/magical-pdf.
  4. Если Codex спросит подтверждение, подтвердите открытие этой папки.
  5. Напишите Codex такую фразу:
Продолжи работу над Magical PDF. Сначала проверь git status, затем помоги с задачей.
Если перед этим вы запускали start-macos.command, Codex откроет уже свежую версию проекта.

6. Как продолжить работу с Windows ПК или другого Mac

  1. На первом компьютере попросите Codex сохранить работу в GitHub.
  2. Убедитесь, что Codex написал, что push выполнен.
  3. На другом компьютере откройте папку magical-pdf.
  4. Запустите стартовый скрипт для этого компьютера: start-macos.command на Mac или start-windows.cmd на Windows.
  5. Откройте эту же папку в Codex.
  6. Продолжайте работу.

Готовая фраза для Codex после завершения задачи:

Сохрани эти изменения в GitHub в общий проект Magical PDF.

7. Как запустить WEB-версию для проверки

  1. Откройте папку magical-pdf.
  2. Откройте Terminal в этой папке.
  3. Выполните команду:
npm run dev:web

Откройте в браузере:

http://127.0.0.1:5173/

8. Как собрать macOS-приложение

  1. Откройте Terminal в папке magical-pdf.
  2. Выполните команду:
npm run build:mac

Готовые файлы появятся здесь:

src-tauri/target/release/bundle/
Файлы .app и .dmg не добавляйте в Git. В Git хранятся исходники, а сборки создаются отдельно.

9. Как открыть iOS-проект

  1. Откройте Terminal в папке magical-pdf.
  2. Выполните команды:
npm run prepare:ios
npm run open:ios

После этого откроется Xcode. В Xcode выберите подпись, устройство или simulator.

10. Что делать, если что-то пошло не так

  • Если Codex видит старую версию проекта, запустите scripts/start-macos.command ещё раз.
  • Если macOS запрещает запуск .command, откройте Terminal и выполните chmod +x scripts/start-macos.command.
  • Если npm не находится, установите Node.js LTS.
  • Если iOS-сборка не идёт, проверьте Xcode, CocoaPods и Apple Developer signing.

11. Самая короткая памятка

1. Открыть Documents/magical-pdf/scripts
2. Запустить start-macos.command
3. Открыть Codex
4. Выбрать папку Documents/magical-pdf
5. Написать Codex, что нужно сделать
6. В конце попросить Codex сохранить изменения в GitHub

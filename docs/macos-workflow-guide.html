Magical PDF: работа на macOS

Главная идея: GitHub хранит общий проект, а Mac берёт последнюю версию из GitHub и отправляет туда законченные изменения.

1. Что где хранится

Общий репозиторий проекта:

https://github.com/beaver20007/magical-pdf

В одном репозитории лежат исходники WEB, macOS, iOS и Windows версий. Готовые сборки .dmg, .app, .exe и временные папки сборки в Git не кладём.

2. Один раз подготовить Mac

  1. Установите Xcode из App Store.
  2. Откройте Xcode один раз, примите license и дайте ему установить компоненты.
  3. Установите Node.js LTS: https://nodejs.org/
  4. Установите Rust: https://www.rust-lang.org/tools/install
  5. Для iOS-сборки установите CocoaPods, если он ещё не установлен.
sudo gem install cocoapods
Для финальной iOS-сборки на iPhone понадобится Apple Developer account и корректная подпись в Xcode.

3. Первый раз скачать проект

Откройте Terminal и выполните:

cd ~/Documents
git clone https://github.com/beaver20007/magical-pdf.git
cd magical-pdf
npm ci

4. Каждый раз перед началом работы

Перед изменениями всегда подтягивайте свежий main:

cd ~/Documents/magical-pdf
git switch main
git pull origin main
npm ci
Если git status ничего не показывает, локальный проект чистый и совпадает с GitHub.

5. Запустить WEB-версию

npm run dev:web

Откройте:

http://127.0.0.1:5173/

6. Собрать WEB-версию

npm run build:web

Результат появляется в папке dist/. Эта папка временная и в Git не добавляется.

7. Собрать macOS-приложение

npm run build:mac

Готовые macOS-файлы появятся внутри:

src-tauri/target/release/bundle/

Обычно там будут .app и .dmg. Их не нужно коммитить в Git.

8. Подготовить и открыть iOS-проект

npm run prepare:ios
npm run open:ios

После этого откроется Xcode workspace. В Xcode выберите Team, Bundle Signing и устройство или simulator.

Если менялась WEB-часть приложения, перед iOS-проверкой снова выполняйте npm run prepare:ios, чтобы обновить файлы внутри iOS-проекта.

9. Как сохранить свою работу в GitHub

Простой вариант для небольших изменений

git status
git add .
git commit -m "Короткое описание изменения"
git push origin main

Аккуратный вариант для заметной задачи

git switch -c feature/short-task-name

# внесите изменения и проверьте приложение

git status
git add .
git commit -m "Describe the change"
git push -u origin feature/short-task-name

После push откройте Pull Request на GitHub и объедините его в main.

10. Как продолжить с другого компьютера

  1. На текущем Mac закончите работу: git status, git add, git commit, git push.
  2. На другом компьютере откройте проект.
  3. Выполните git pull origin main.
  4. Продолжайте работу с последней общей версии.

11. Быстрая диагностика

git status
node --version
npm --version
rustc --version
cargo --version
xcodebuild -version

Если команда не находится, перезапустите Terminal и проверьте установку соответствующего инструмента.

12. Главное правило

Перед работой: git pull origin main. После законченной работы: git commit и git push. Тогда можно спокойно продолжать проект с Mac, Windows ПК или другого компьютера.

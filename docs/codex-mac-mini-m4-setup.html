Codex на Mac Mini M4: установка настроек и коннекторов

Инструкция для нового Mac Mini M4: установить Codex, перенести глобальные настройки коннекторов и подготовить подключение оставшихся сервисов через API/MCP.

1. Что получится в итоге

  • Codex будет установлен и открыт на Mac Mini M4.
  • В глобальном конфиге ~/.codex/config.toml будут включены Google Drive, Gmail, Google Calendar, Slack, Notion, Linear и Teams.
  • Google/Slack/Notion/Linear будут проверены через безопасные read-only действия.
  • Для оставшихся сервисов будет понятный порядок: где взять API key, OAuth credentials или webhook URL.

2. Что подготовить заранее

  1. Mac Mini M4 с доступом в интернет.
  2. Логин и пароль от того же аккаунта Codex/OpenAI.
  3. Доступ к Google-аккаунту beaver20007@gmail.com, если Codex попросит повторную авторизацию.
  4. Доступ к Slack, Notion и Linear, если они попросят OAuth-подтверждение.
  5. GitHub-репозиторий с папкой codex-setup. Сейчас она лежит в проекте magical-pdf.

3. Установить базовые инструменты на Mac

  1. Установить Codex Desktop.
  2. Войти в Codex под тем же аккаунтом Codex/OpenAI.
  3. Открыть приложение Terminal.
  4. Проверить Git:
git --version

Если macOS предложит установить Command Line Tools, согласиться и дождаться окончания установки.

4. Скачать репозиторий с настройками

Важно: файлы codex-setup и эта PDF/HTML-инструкция появятся на Mac Mini M4 только после того, как текущие изменения будут закоммичены и отправлены в GitHub с основного ПК.

Перед началом на Mac Mini M4 на основном ПК нужно попросить Codex:

Закоммить и отправь в GitHub папку codex-setup и инструкции по настройке Codex-коннекторов.

Пока codex-setup лежит внутри проекта magical-pdf, самый простой путь такой:

cd ~/Documents
git clone https://github.com/beaver20007/magical-pdf.git
cd magical-pdf/codex-setup
Если позже папка codex-setup будет вынесена в отдельный GitHub-репозиторий, нужно будет клонировать уже его и перейти в его папку.

5. Запустить настройку коннекторов

В Terminal выполнить:

chmod +x scripts/setup-codex-connectors-macos.sh
chmod +x scripts/verify-codex-connectors-macos.sh
./scripts/setup-codex-connectors-macos.sh

Скрипт сделает резервную копию ~/.codex/config.toml и добавит только недостающие секции плагинов.

Ожидаемые секции:

[plugins."google-drive@openai-curated"]
enabled = true

[plugins."gmail@openai-curated"]
enabled = true

[plugins."google-calendar@openai-curated"]
enabled = true

[plugins."slack@openai-curated"]
enabled = true

[plugins."notion@openai-curated"]
enabled = true

[plugins."linear@openai-curated"]
enabled = true

[plugins."teams@openai-curated"]
enabled = true

6. Перезапустить Codex и авторизовать сервисы

  1. Полностью закрыть Codex.
  2. Открыть Codex заново.
  3. Открыть Settings -> Connectors.
  4. Проверить Google Drive, Gmail, Google Calendar, Slack, Notion, Linear и Teams.
  5. Если рядом с сервисом есть кнопка Connect/Auth, нажать ее и пройти OAuth-авторизацию.
Пароли и токены не нужно отправлять Codex сообщением. OAuth нужно проходить только в официальном окне авторизации сервиса.

7. Проверить конфиг на Mac

В папке codex-setup выполнить:

./scripts/verify-codex-connectors-macos.sh

Если все хорошо, скрипт покажет [enabled] для каждого плагина.

8. Команда для Codex на новом Mac

После установки попросите Codex выполнить настройку самостоятельно:

Прочитай файл codex-setup/CODEX_REMOTE_INSTRUCTIONS.md и настрой глобальные коннекторы Codex на этом Mac. 
Запусти macOS-скрипты из codex-setup/scripts, проверь ~/.codex/config.toml, затем безопасно проверь доступность Google Drive, Gmail, Google Calendar, Slack, Notion и Linear. 
Не отправляй письма, не создавай события и не меняй данные без моей явной просьбы.

9. Как Codex должен проверять подключенные коннекторы

10. Microsoft-блок на Mac

  1. Создать личный Microsoft-аккаунт на https://signup.live.com/.
  2. Можно использовать существующий Gmail-адрес, новый Outlook-адрес не обязателен.
  3. Один раз войти на https://account.microsoft.com/.
  4. Один раз открыть https://outlook.live.com/ для инициализации Outlook.
  5. Один раз открыть https://teams.microsoft.com/ для инициализации Teams.
  6. В Codex открыть Settings -> Connectors и подключить Teams, Outlook Email и Outlook Calendar, если они доступны.
  7. SharePoint может потребовать рабочий или учебный Microsoft 365-аккаунт.

11. Оставшиеся коннекторы: общий порядок

  1. Войти в нужный сервис.
  2. Создать API key, personal access token, OAuth app или webhook URL.
  3. Выдать минимальные права: сначала read-only, потом расширять при необходимости.
  4. Передать секрет Codex только через безопасный механизм секретов или менеджер паролей.
  5. Попросить Codex добавить MCP/API-конфигурацию и выполнить один read-only тест.
Никогда не коммитить API keys, client secrets и webhook secrets в GitHub.

13. Рекомендуемый порядок добавления оставшихся сервисов

  1. Сначала простые API-key сервисы: Asana, Apollo.io, MailerLite, Intercom, HubSpot.
  2. Потом Stripe в test mode с restricted key.
  3. Затем automation hubs: n8n, Make, Zapier.
  4. После этого специальные или тарифно-зависимые сервисы: Canva, Gamma, Granola, Clay Enterprise.

# Magical PDF

Прототип приложения для преобразования PDF в JPEG-страницы и последующей сборки этих страниц обратно в новый PDF. Цель - сделать PDF с факсимиле, печатями или вставленными картинками похожим на скан документа.

## Запуск

```bash
npm run dev:web
```

Открыть:

```text
http://127.0.0.1:5173/
```

## Web-сборка для хостинга

Собрать статическую версию:

```bash
npm run build:web
```

На хостинг нужно загрузить содержимое папки `dist`.

Особенности Web-версии:

- Все библиотеки лежат локально в `dist/vendor`.
- PDF не отправляется на внешний сервер.
- В современных Chromium-браузерах кнопка скачивания открывает выбор места сохранения.
- Если браузер не поддерживает выбор места сохранения, используется стандартная загрузка браузера.
- Для локальной проверки в Codex `npm run dev:web` дополнительно умеет сохранять файлы в `Downloads`, если встроенный браузер ломает обычную загрузку.


## Текущий MVP

- Работает полностью в браузере.
- Обрабатывает файлы локально на устройстве; отправки PDF на сервер нет.
- Использует локальные копии библиотек, поэтому установленные приложения могут работать без интернета.
- Преобразует каждую страницу PDF в JPEG без изменения размера страницы.
- Собирает новый PDF из JPEG-страниц с тем же визуальным расположением.
- Экспортирует JPEG-страницы высокого разрешения в ZIP-архив.
- Поддерживает три режима качества страниц: низкое, среднее и высокое. Каждый режим автоматически задает разрешение и качество JPEG.

## Приложение для macOS

В проект добавлена оболочка Tauri для macOS-приложения, которая использует тот же интерфейс.
После установки приложение работает офлайн: все библиотеки лежат внутри сборки.
Для локальной сборки нужен Rust/Cargo.

Один раз установить зависимости:

```bash
npm install
```

Запустить веб-сервер в одном терминале:

```bash
npm run dev:web
```

Запустить macOS-приложение в другом терминале:

```bash
npm run dev:mac
```

Собрать `.app` и `.dmg`:

```bash
npm run build:mac
```

## Windows / ПК

Собрать Windows-инсталлятор `.exe` через Tauri:

```bash
npm run build:windows
```

Обычно `.exe` нужно собирать на Windows-машине или в Windows CI. macOS не всегда может корректно собрать Windows-инсталлятор без отдельной cross-compilation настройки.

## Установщики через CI

В проект добавлен GitHub Actions workflow:

```text
.github/workflows/build-installers.yml
```

Он собирает Web-версию, macOS `.dmg` и Windows `.exe` на подходящих runner-ах.

## Android и iOS

Для мобильных приложений добавлена конфигурация Capacitor. Она использует папку `dist`, куда попадают все файлы интерфейса и локальные библиотеки.
На iOS готовый PDF или ZIP сохраняется через системное меню iOS: можно выбрать `Сохранить в Файлы`, AirDrop, почту и другие действия.

После установки зависимостей:

```bash
npm install
```

Подготовить мобильные проекты:

```bash
npm run add:android
npm run add:ios
npm run sync:mobile
```

Открыть Android-проект:

```bash
npm run open:android
```

Открыть iOS-проект:

```bash
npm run open:ios
```

После установки на устройство приложение должно работать без доступа к интернету.

Подробная инструкция по iOS находится в:

```text
IOS_SETUP.md
```

## План развития продукта

- Web: текущий браузерный движок, доступный через сайт с интернетом.
- ПК и macOS: оболочка Tauri вокруг того же интерфейса, все библиотеки внутри приложения, работа офлайн.
- Android и iOS: Capacitor или нативная обертка, все библиотеки внутри приложения, работа офлайн.
- Ядро: общие настройки конвертации и тесты, чтобы все платформы работали одинаково.
